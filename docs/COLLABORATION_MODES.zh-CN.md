# 协作方式与工作模式

Reames Agent 有两条彼此独立的运行轴：

1. **协作方式**：普通、计划、目标，决定“怎样和你协作”。
2. **工作模式**：经济、均衡、交付，决定“怎样分配上下文、工具面和完成强度”。

工具权限（询问、自动、Yolo）是第三条独立轴，只决定审批策略，不会改变协作方式或工作模式。

## 协作方式

### 普通

直接回答或执行当前请求。适合边界清楚的问题、小改动和日常对话。

### 计划

先只读分析并给出计划，确认后再执行写文件、命令、提交或发布等有副作用的操作。桌面端可在输入框菜单中选择“计划”，CLI/TUI 可用 `Shift+Tab` 切换。

适合跨模块改动、风险评估、方案尚未确定或需要先审阅实施步骤的任务。计划模式不是自动完成任务；它会在执行前等待确认。

### 目标

围绕明确目标持续推进，直到完成、真正阻塞、被停止或需要用户拥有的关键决策。复杂目标建议包含范围、成功标准、限制和暂停条件；模板见 [`TASK_CONTRACT.zh-CN.md`](./TASK_CONTRACT.zh-CN.md)。

明显属于长周期研究、排障或优化的目标会启用持久化 AutoResearch 状态：

```text
.reames-agent/autoresearch/<task-id>/
```

可用 `/goal --research <目标>` 强制启用，或 `/goal --simple <目标>` 保持轻量 Goal。目标模式不会绕过审批、沙箱或证据门禁。

## 工作模式

### 经济（economy）

精简初始工具 schema 和技能索引；Skills、MCP、LSP/CodeGraph、`web_fetch`、安装源和子代理等可选能力按需连接。它降低固定上下文开销，但首次启用某类能力可能多一个衔接步骤。

适合日常问答、代码解释、小范围读取和多数轻量修改。它不会降低模型本身的推理能力，也不会强制缩短最终回答。

### 均衡（balanced，默认）

启动时提供完整常用工具面，在质量、速度和成本之间采用常规取舍。旧会话中持久化的内部值 `full` 会自动投影为 `balanced`，无需迁移。

适合大多数开发任务，也是不确定如何选择时的默认项。

### 交付（delivery）

优先得到经过验证的完整结果：明确验收标准、检查相关代码与项目规则、修复根因、运行聚焦验证、审阅 diff 和相邻行为，并在没有真正阻塞时继续收尾。交付模式复用现有 Todo、`complete_step`、project checks、evidence、checkpoint 和子代理交付机制，不建立第二套 Agent，也不放宽权限或沙箱。

适合复杂实现、可靠性修复、发布前收口和“完成后再汇报”的任务。它通常比均衡模式消耗更多模型调用和验证时间。

## 如何切换

- **Desktop**：输入框左下角菜单分别选择协作方式和工作模式；经济或交付模式会显示可点击的状态标签，点击后返回均衡。
- **一次性 CLI**：`reames-agent run --profile economy|balanced|delivery ...`
- **交互 TUI**：启动时使用 `--profile`，会话内使用 `/work-mode economy|balanced|delivery`。
- **Serve**：`reames-agent serve --profile ...`；模型/effort 重建会保留启动工作模式。
- **ACP**：`reames-agent acp --profile ...` 设置默认值；客户端可通过 `work_mode` session config option 切换。
- **Bot/Gateway**：当前固定默认均衡。只有建立稳定的 per-connection 配置与持久化合同后才开放渠道级切换。

省略 `--profile` 恢复会话时，会继承会话保存的工作模式；新会话默认均衡。活动 turn、待确认问题或后台任务存在时，运行时切换会拒绝或按 ACP 合同排队，避免半切换。

## 组合关系

| 组合 | 是否支持 | 说明 |
| --- | --- | --- |
| 普通/计划/目标 + 经济 | 支持 | 协作流程不变，初始工具面按需加载。 |
| 普通/计划/目标 + 均衡 | 支持 | 默认完整工具面。 |
| 普通/计划/目标 + 交付 | 支持 | 在相同协作流程上提高验证与收尾强度。 |
| 计划 + 目标 | 不同时启用 | 二者属于同一协作轴，切换会替换当前协作方式。 |
| 任一模式 + 询问/自动/Yolo | 支持 | 工具权限只控制审批，不改变工作模式。 |

工具权限详见 [`TOOL_APPROVAL_MODES.zh-CN.md`](./TOOL_APPROVAL_MODES.zh-CN.md)。

## 推荐选择

- 日常开发：普通 + 均衡 + 询问/自动。
- 低开销阅读或小任务：普通 + 经济。
- 先审方案：计划 + 均衡；需要完整能力时不必开启经济。
- 长线持续推进：目标 + 均衡或交付。
- 发布前、复杂可靠性修复：目标/普通 + 交付。
