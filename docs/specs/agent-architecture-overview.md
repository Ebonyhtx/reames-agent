## 核心模块职责

| 模块 | 职责 | 来源 |
|:---|:---|:---|
| **对话循环** | 用户输入 → 调 LLM → 执行工具 → 返回结果 | Hermes（深度修改） |
| **记忆系统** | 4 层记忆管道、符号化短期记忆、混合检索 | TencentDB（接入） |
| **工具系统** | 85+ 内置工具 + 插件 + MCP | Hermes（扩展） |
| **钩子系统** | PreToolUse/PostToolUse/Stop 三层强制 | Reasonix（移植） |
| **缓存优化** | 字节稳定前缀策略 | Reasonix（移植） |
| **多平台** | 34 个消息平台适配器 | Hermes（现有） |
| **定时任务** | cron 调度、自动化维护 | Hermes（现有） |
| **熵管理** | 技能质检、记忆修剪、插件检查、配置漂移 | **原创** |
| **MCP 集群** | SearXNG、GitHub MCP、爬虫、文档解析等 | 社区生态 |

## 技术栈

| 层 | 语言 | 关键依赖 |
|:---|:---|:---|
| Agent 核心 | Python | OpenAI SDK, FastMCP, anyio |
| 记忆系统 | TypeScript | SQLite, sqlite-vec, BM25 |
| 桌面端 | TypeScript | Electron, React |
| 搜索 | Python | SearXNG, FastMCP |
| 插件 | Python | Hermes Plugin API |

## 开发路线

```
阶段2（当前）→ 设计定稿 → 写设计文档 → 你审阅
阶段3 → 记忆系统集成（TencentDB）
阶段4 → MCP 服务部署（SearXNG + GitHub + 代码分析）
阶段5 → 原生插件开发（Git/编译/代码分析/爬虫）
阶段6 → 钩子系统移植（Reasonix）
阶段7 → 熵管理系统（你的创新）
```
