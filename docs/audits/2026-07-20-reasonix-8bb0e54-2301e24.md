## MCP stdio 与 Desktop 生命周期

`internal/plugin/transport_stdio.go` 现在区分 notification 与 server request：回答 `ping`，未知 method 返回
JSON-RPC method-not-found；reply 进入独立有界队列和专用 writer，读 stdout 的 goroutine不会因 stdin 背压或
共享写锁形成四方死锁。普通 client 写串行化，EOF 前最后一行仍会处理，reply 写失败不会伪装成整个 read
transport 已死亡。定向普通/race 测试覆盖 server ping、blocked reply writer、最后一行和取消。

Desktop 不复制 Reasonix 私有 `turnStartMu`，而是复用 Reames 已有的 `pluginRuntimeGate` 和 Controller
`BeginRuntimeMutation`：

```text
plugin/turn admission writer
  -> async controller-build admission writer
  -> runtimeRebuildMu
  -> every visible + detached Controller BeginRuntimeMutation
  -> App.mu / shared Host / per-controller Registry / config
```

所有 Wails MCP mutation（add/update/remove/reconnect/reverify/clear-auth/reader-trust/enable/tier）进入同一
合同。mutation 前作废已经按旧 generation 启动的异步 build，mutation 期间禁止新 build admission，结束后才
重启仍为空的 tab。插件 install/update/remove/enable/disable 使用相同顺序，并在 package mutation 前作废旧
build，避免 uninstall 期间发布一个仍持有旧插件/MCP surface 的新 Controller。所有可见与 detached Controller
都被 reservation；兄弟标签页的 foreground turn、pending prompt 或 background job 会在任何配置/Host 变化前
拒绝 mutation。

回归覆盖：

- 十个 MCP mutation entry point 都会因 sibling runtime work fail closed；
- mutation reservation 期间 sibling turn 不能进入，release 后恢复 liveness；
- late controller build 等待 admission writer；
- plugin apply callback 开始前旧 startup generation 已取消；
- shared Host 的 active-tab disable 不删除 sibling registry/client；
- plugin disable 同时撤销 visible 与 detached runtime；
- focused normal 与 race 通过，Desktop Go 全量在生产实现稳定后通过。

## 中断轮次恢复

`provider.Message` 新增本地元数据 `LocalOnly`、`InterruptedTurn`、`CreatedAt`。中断时：

- partial assistant text、reasoning 与未闭合 tool args 可以持久化供用户查看，但在 Provider request 边界剥离；
- 完整 assistant tool-call / tool-result 对在优雅停止和 Provider 错误时保留；
- legacy 孤立 tool 片段通过 sentinel 丢弃，不能重新进入模型；
- 下一条真实 user turn 只收到有界、结构化、HTML 转义的 recovery fact；
- recovery fact 不进入标题、preview、普通用户显示、compaction summary 或显式 range summary；
- compaction 不折叠当前 active turn，prune 不误删 display-only recovery records；
- CLI/Desktop history 显示 partial text、reasoning、tool cards 和明确中断 notice。

与 Reasonix 不同，Reames 对真正进程崩溃/冷启动保留 checkpoint rollback：未知副作用不会因为存在 partial
display 而被当成完成。该边界与 M6 outbound obligation 不冲突；前者是本地模型轮次 transcript，后者是成功
turn 最终文本的渠道投递事务。

## 明确不采用

- 不删除 Reames 的 MCP identity receipt、launcher pin、project provenance 或显式 reader trust；
- 不把 user-configured MCP 自动授权扩大为 destructive tool 的永久免审；
- 不因 Remote SSH 修复存在就提前开启 SSH 产品面；
- 不把任意中断输出、未闭合工具调用或 crash 前未知副作用重新喂给 Provider；
- 不引入第二套 Agent、Desktop 或插件 runtime。
