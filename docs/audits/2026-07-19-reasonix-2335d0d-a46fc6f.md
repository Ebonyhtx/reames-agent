## 已采用

### 每模型 context window（`c77af74c`）

- `ProviderModelOverride.ContextWindow` 贯通 TOML render/round-trip、模型解析、
  Desktop DTO 和 React 设置页。
- 模型覆盖为正数时覆盖 provider 默认窗口；负数规范化为继承值。
- UI 显示继承提示，并对低于 16384 tokens 的显式值给出警告。
- 后台模型发现、手工候选和删除模型的合并逻辑不会丢失其他 override 字段。

### 有界会话保存锁与退出恢复（`dd929bf2`）

- Unix `flock` 与 Windows `LockFileEx` 改为非阻塞尝试 + 五秒有界重试。
- 临时竞争在 deadline 前释放时正常保存；永久持锁返回
  `ErrSessionFileLockHeld`，不再无限冻结导航或退出。
- `SnapshotForShutdown` 仅在该 sentinel 上绕过原文件，写入带 process writer
  identity 的 recovery branch，保留普通 session/event/meta 格式。
- Desktop 退出先完成 shutdown snapshot/recovery，再关闭 Controller 和释放 lease。
- recovery handoff 的 lease-backed tab path 使用原子镜像，防止并发 layout save
  用 Controller 的旧路径覆盖已提交 recovery path。

### Windows `.cmd/.bat` Hook（`b9890356`、`1a8b6f82`）

- 普通用户/全局 Hook 对简单、已引号包围的 batch path 使用精确 `cmd.exe`
  command line，保留参数文本并拒绝命令链、换行和不完整引号。
- package-owned Hook 不绕过 Reames 的严格 OS sandbox。它在 sandbox argv 内使用
  PowerShell `-EncodedCommand` 调用 batch path，避免 `cmd.exe` 引号歧义和路径
  插值，同时保留最小环境、forbid-read 和进程树回收。
- Windows 真实 E2E 同时覆盖普通带空格路径与 sandboxed package batch Hook。

### 测试用户状态隔离（`bb9c4aa5`、`df1578bc`、`9c96fd7a`）

- 新增 `internal/testenv`，统一隔离 HOME/USERPROFILE、XDG config/cache/state、
  AppData/LocalAppData、TEMP/TMP/TMPDIR。
- 清除继承的 `REAMES_AGENT_HOME`、`REAMES_AGENT_STATE_HOME`、
  `REAMES_AGENT_CACHE_HOME`，防止测试写入调用者的真实配置和缓存。
- boot、bot、botruntime、config、control、CLI、plugin、repair、serve、skill、
  gatewaysetup、desktop 等状态写入包接入 process-level guard；helper subprocess
  复用父进程隔离环境，不创建越界 home。
- config 测试固定 UserConfig、cache、workspace lease 和 managed worktree 均位于
  隔离根下。本实现额外隔离 TEMP/TMP，直接降低测试留下大量 C 盘小文件的风险。

### 本地运行可靠性（`26aa92cb`）

- Mermaid pan/zoom 在 hidden/zero-size layout 时延迟同步，并捕获 singular matrix
  异常；ResizeObserver 和 RAF cleanup 有界。
- `Object.hasOwn` polyfill 在 React/markdown import 前加载，兼容旧 WebKit。
- credential key/value redaction 从复杂 regexp 改为线性 scanner，保留
  Authorization scheme，避免长 transcript 并发脱敏出现高开销或泄漏。
- 明确拒绝 Reasonix crash-report worker、远端上传和遥测；Reames 按用户要求只保留
  本地诊断。
- TanStack `directDomUpdates/containerRef` 未采用：Reames 当前
  `@tanstack/react-virtual` 公共类型没有该 API，机械复制会形成未受支持的私有合同。

### Windows-safe session filename（`1e1ba41d`）

`NewSessionPath` 除斜杠外继续替换 `:< >"|?*` 等 Windows 非法字符。真实
`nemotron-3-nano:30b` 形状覆盖保存、sidecar 基础路径和 stat，避免 ADS/lease
路径异常。

### 最大化窗口 resize 光标恢复（`a46fc6f4`）

- Windows frameless titlebar 的 native maximized 状态只查询一次并共享给窗口按钮与
  `useWailsResizeFix`，mousemove 热路径不执行 IPC。
- 最大化启动或切换时立即清除 stale `resizeEdge`，恢复 Wails 记住的默认 cursor；
  普通近全屏/FancyZones 窗口不根据尺寸猜测为最大化。
- 双击标题栏与按钮切换后均重新同步 native 状态，并使用 generation 防止迟到的
  promise 覆盖较新状态。
- 17 项 JSDOM 回归覆盖 disabled、stale startup、near-full normal、maximize/restore
  transition、edge detection 与 cleanup。

## 明确延后

### 取消轮次显示缓冲（`2b43e0f8`）

Reames 当前 canonical session 已在取消时删除 assistant/tool 残余并保留真实用户
提示，持久模型历史是安全的；但尚未把本轮部分 reasoning/tool 输出另存为只读
display history。该功能需要统一 executor/planner display buffer、跨进程 sidecar
锁、detached/reattached runtime 和删除/垃圾回收合同，不能把上游 Desktop 状态机
直接复制进现有 `plannerDisplay`。它登记为后续 Desktop session-display 可靠性任务，
不得宣称已采用。

### CLI 中键 PRIMARY/tmux 粘贴（`44115605`）

现有 Reames 已有右键文本粘贴、SSH 本地剪贴板边界、长文本折叠和统一
`tea.PasteMsg`。中键 PRIMARY/tmux 是有价值的 Linux/TUI UX 增量，但不是当前
M6/会话可靠性交付门槛；后续采用时必须复用 `processpolicy.ProcessEnvironment()`、
隐藏 composer、SSH 和 selection-byte 合同。

### 可配置会话宽度（`4c1702ca`）

标准 960px / 90% full width 属于 P5/P9 Desktop 体验候选。当前批不增加新的
appearance 配置状态，避免与 Theme Pack、responsive layout 和 accessibility
验证混在一起。
